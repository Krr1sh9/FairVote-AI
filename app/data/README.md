# Synthetic example data

The CSV files in this folder are synthetic demonstration datasets generated by the FairVote-AI experiment tools. They are included so the dashboard can be tested without collecting real polling data.

Some synthetic files contain columns such as `true_choice` or `stated_choice`. These columns are **evaluation-only**: they exist so simulations can measure estimator error against known ground truth. Real respondent exports from `respondent/server.py` should not contain true votes, and real polling mode does not require `true_choice`.

For real deployments, use the privatized reported-answer column and demographic fields only.

The presence of `true_choice` in these synthetic files does not change the privacy protocol: the respondent server still rejects true-answer submissions, and the neural model trains on privatized reported labels.
