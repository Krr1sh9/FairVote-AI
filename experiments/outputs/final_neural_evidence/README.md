# Final Neural MRP Evidence Pack

This folder contains the strongest completed neural-MRP experiment outputs available for the dissertation submission package.

**Evidence status:** computationally constrained final-style evidence, not the full preset.

The preferred full preset was attempted but did not complete in the available execution environment. This evidence pack therefore uses a constrained configuration that still covers all epsilon values, all bias scenarios, all implemented estimator methods, and the expected result metrics. These outputs should be cited as computationally constrained evidence, not as exhaustive full-preset results.

## Command used

```bash
python -m experiments.evaluate_neural_mrp \
  --preset full \
  --eps 0.2,0.5,1.0,2.0 \
  --sample_sizes 500,1000 \
  --scenarios no_bias,nonresponse,shy_voter,privacy_helps \
  --population_n 5000 \
  --trials 1 \
  --mrp_steps 5 \
  --neural_steps 5 \
  --neural_hidden_layers 16 \
  --mrp_batch_size 512 \
  --neural_batch_size 512 \
  --out_dir experiments/outputs/final_neural_evidence_reproduced
```

> Note: the original generation path was environment-specific. To reproduce locally, use the relative output path shown above. The included checked-in evidence remains in `experiments/outputs/final_neural_evidence/`; use a separate reproduction folder to avoid overwriting it.

## Headline result summary

Values below are computed from `summary.csv`. Lower L1/runtime is better; higher winner correctness is better.

| Method | Mean overall L1 ↓ | Winner correctness ↑ | Mean runtime sec ↓ |
|---|---:|---:|---:|
| `raw_reported_distribution` | **0.163** | 0.250 | 0.000 |
| `baseline_rr_debias` | 0.430 | 0.250 | 0.001 |
| `mrp_rr_poststrat` | 0.176 | 0.344 | 0.005 |
| `neural_rr_mrp` | 0.204 | 0.125 | 0.030 |
| `mrp_misreport_rr_poststrat` | 0.211 | **0.906** | 0.768 |
| `mrp_learned_misreport_rr_poststrat` | 0.229 | 0.094 | 0.006 |

The raw reported distribution has the lowest average overall L1 in this constrained synthetic run, but it is an uncorrected descriptive baseline over privatized reports, not an RR-corrected estimator of the latent true distribution. Among the RR-aware corrected/model-based estimators, linear RR-aware MRP (`mrp_rr_poststrat`) has the lowest average overall L1. Neural RR-MRP is therefore valid as a privacy-compatible AI estimator, but this evidence pack does not show that it is generally superior to linear RR-aware MRP.

## Known limitations

These outputs are not the exhaustive full-preset final run.

Important limitations:

1. `trials = 1`, so the results do not provide stable variance estimates.
2. `mrp_steps = 5` and `neural_steps = 5`, so learned models are undertrained compared with the intended full preset.
3. `population_n = 5000`, smaller than the preferred full-preset population.
4. Sample sizes cover `500` and `1000`, not the full intended set including `5000` and `10000`.
5. These outputs should be used as computationally constrained evidence and pipeline validation, not as proof that neural RR-MRP is generally superior.
6. Neural RR-MRP should not be described as automatically better. Any claim should be based on the values in `summary.csv`, `neural_comparison.csv`, `method_rankings.csv`, and `neural_verdict.csv`.

## Recommended dissertation wording

Use wording like:

> A computationally constrained neural-MRP evidence run was completed using all four epsilon values and all four bias scenarios, with two sample sizes and one trial per condition. The run verifies the full experiment pipeline and provides indicative results, but it is not an exhaustive full-preset evaluation. Because the learned models used reduced training steps, the results should be interpreted cautiously and not treated as proof that neural RR-MRP is universally superior.

Avoid wording like:

> The full neural MRP experiment proves that the AI model is better.

That claim is not supported by this evidence pack.
